# Networking Action Card (v4.5)

## Primary Objective
Gain signal on entering VC.

---

# DAILY OPERATING FLOW

## 1️⃣ Identify State
Every contact must be in one state:

- legibility_sent
- value_ping_sent
- question_sent
- reprompt_sent
- engaged
- call_ready
- warm_network
- closed_dormant

Find state → execute next action → log event.

---

# GIVE–GIVE–ASK RULE

Before asking for a call:

✔ At least 1 proactive give  
    (value_ping_sent OR public_comment)

✔ At least 1 conversational give  
    (meaningful insight during engagement)

Then call_ask_sent.

Never ask from cold.

---

# TIMING GUARDRAILS

Legibility → wait 5 working days  
Value ping → wait 5 working days  
Question → wait 10 working days  
Reprompt → wait 5 working days  

Respond to engagement within 24–48h.

One reprompt maximum.

---

# TIER DEPTH LIMITS

Tier A  
Full ladder allowed.

Tier B  
Legibility → Value → Question  
Reprompt only if strong relevance.  
Call ask only after clear engagement.

Tier C  
Legibility only.  
No mechanical follow-up.

---

# WHEN TIME IS LIMITED

Priority order:

1. Engaged Tier A
2. Tier A question_sent / reprompt_sent
3. Tier B value/question
4. New Tier A connections
5. Warm network

Never let Tier C consume mechanical bandwidth.

---

# WEEKLY CHECK

Review:

- calls_completed_per_legibility
- calls_completed_per_call_ask
- Tier distribution (A/B/C/U)
- proactive_gives_per_legibility

Do not redesign mid-cycle.

---

# STOP RULES

Two unanswered asks → close.  
Silence from start → close silently.  
Explicit no → close immediately.

---

# REMEMBER

You are building signal, not chasing outcomes.

Consistency > optimisation.
